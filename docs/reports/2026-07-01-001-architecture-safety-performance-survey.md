## Executive summary

1. **Stop the line first: swift-tui main is red.** A new flaky test (`TaskReadsUnbodiedStateTests`) fails the only required child gate on ~62% of runs since `a210b7be`, it is unregistered in the flake ledger, and two org pin bumps (`96a4f4d`, `b08c94a`) were made over the red because `//:org_fast` runs no native tests and never checks child CI status (F01, CONFIRMED critical).
2. **The ground truth in prior reports and session memory is materially stale — in the good direction.** FrameworkStress known issues are **zero** at HEAD (not 16); multi-task-per-node **landed** (`a8c416e9`); #4 pipeline-IR encapsulation **landed** (`315fd53c`, `ResolvedNode` is `package`); the FrameScheduler race, CSI overflow, offset-cull corruption, and env-reader aliasing were all fixed silently. The 15-item + supplemental #16–#31 program is effectively complete (§Ground truth).
3. **The dominant bug family — identity churn across reconciliation seams — is now fully mapped, and its root fix is ~78% built.** Every hard bug of the past month is one two-node transient re-surfacing per registry. HEAD compensates with a five-mechanism teardown/keep stack plus route-side patches; entity routing ("same entity ⇒ same ViewNode") already exists for `.id(Hashable)` nodes. Finishing Stage 6 and deleting the compensator stack is the highest-leverage strategic investment in the codebase (F02).
4. **The safety-net has a release-shaped hole.** No release-configuration test lane exists anywhere; the sampled-release soundness probe has never executed in CI; the raster mismatch oracle silently repairs and records nothing; no oracle observes subtractive teardown; and a code-traced release-only ghost-trail corruption exists in the incremental damage producer (F05/F06/F07/F04, all CONFIRMED).
5. **Three structural over-invalidations dominate runtime cost:** the portal-root force-queue collapses the dirty frontier to the root on every invalidation-bearing frame (F08); every focus/press change forces two full root re-resolves with reuse suppression (F09); any non-property animation forces whole-app body re-runs every 33ms tick (F32). The infrastructure to narrow all three now exists (single-pass focus-sync, reader indexes, scoped-restore harness).
6. **Pre-1.0 API windows worth spending now:** `Layout.Cache: Sendable` (unshelves the last memory-unsafety-by-construction fix, F11), KeyEvent v2 (Delete/PageUp/F-keys are parsed then dropped; F-keys can inject garbage text via the missing SS3 path, F14), stack layout allocation semantics (SwiftUI divergence with confirmed footguns, F12), and one versioned cross-host wire schema (F18/F19).
