Changes enter as invalidation intent.

          

            State, observation, input, focus, pointer, presentation, and animation
            all converge on the scheduler. The scheduler coalesces causes and
            invalidated identities into a `ScheduledFrame`, then the run loop
            consumes that frame and asks the renderer for new artifacts.
          

        

        

      


      

        
02

        

          
The interactive callpath adds policy around the same products.

          

            A one-shot `DefaultRenderer.render` returns `FrameArtifacts`. An
            interactive session adds input pumping, invalidation coalescing,
            async frame-tail cancellation, completed-frame disposition, focus
            convergence, diagnostics, and presentation to a concrete surface.

    04
    
        
    
          
    FrameArtifacts carry the product bundle.
    
          
    
            The runtime does not hand views to the terminal. It commits a bundle
            of typed products plus diagnostics, presentation hints, and a commit
            plan. Each product owns a different question about the same frame.
          
    
        
    
        
    
      
    

      
    
        
    05
    
        
    
          
    Presentation converts committed data into host output.
    
          
    
            `RunLoop.presentCommittedFrame` selects JSON, accessible output,
            semantic host frames, damage-aware raster presentation, or a raw
            raster surface. Terminal-native hosts plan repaint work and emit
            escape sequences after the renderer has already committed artifacts.
          
    
        
    
        
    
      
    

      
    
        
    06
    
        
    
          
    
            
    On the wire.
    
            
    
              Terminal output is a transport detail below the frame boundary:
              clear/move cursor, incremental row writes, synchronized-output
              wrappers, image protocol replay, and UTF-8 text all flow through
              the presentation surface.
